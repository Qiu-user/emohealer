# 基于AIGC的情绪愈疗医生任务书

## 一、项目概述

### 1.1 项目名称
**基于AIGC的情绪愈疗医生**（EmoHealer AI）

### 1.2 项目背景

**发起背景：**
- **心理健康危机加剧**：后疫情时代，全球焦虑、抑郁等心理问题发病率上升30%以上，中国抑郁症患者超9500万，但专业心理咨询师缺口达130万人
- **传统咨询痛点突出**：心理咨询费用高昂（300-2000元/小时）、资源分布不均、病耻感阻碍求助、服务时间受限
- **AIGC技术爆发**：大语言模型（LLM）在情感计算、自然语言理解方面取得突破，为低成本、规模化心理服务提供技术基础
- **政策利好**：国家卫健委《"十四五"国民健康规划》明确提出推动互联网+心理健康服务

**项目目的：**
构建基于生成式AI的情绪疗愈平台，提供7×24小时个性化心理支持，降低心理健康服务门槛，实现"AI初筛+人工干预"的分级诊疗模式。

**预期解决问题：**
- 缓解专业心理咨询师资源不足问题
- 消除用户寻求心理帮助的心理门槛
- 提供即时、低成本的情绪急救服务
- 建立心理健康数据档案，实现早期预警

### 1.3 目标用户

**核心用户画像：**
| 维度         | 描述                                                         |
| ------------ | ------------------------------------------------------------ |
| **年龄层**   | 18-35岁（Z世代及千禧一代，占比70%），次核心36-50岁职场人群   |
| **性别**     | 女性用户占65%，男性占35%（女性更愿意表达情绪问题）           |
| **职业**     | 大学生（学业压力）、互联网/金融从业者（高压职业）、新手妈妈（产后情绪） |
| **地域**     | 一二线城市为主，三四线城市增长迅速                           |
| **心理特征** | 高敏感人群、社交焦虑、轻度抑郁情绪、职场倦怠、情感困扰       |

**用户需求与痛点：**
- **即时性需求**：深夜情绪崩溃时无法立即找到倾诉对象
- **隐私保护**：不愿让身边人知道自己在寻求心理帮助
- **经济压力**：学生群体和职场新人难以承担长期咨询费用
- **专业门槛**：分不清自己是否需要专业帮助，担心被贴标签
- **持续性差**：传统咨询间隔期内缺乏日常情绪管理工具

## 二、市场调研

### 2.1 行业现状

**发展趋势：**
- **高速增长期**：全球AIGC心理健康市场规模2023年约18亿美元，预计2027年达121亿美元，年复合增长率60%
- **技术融合**：从单一聊天机器人向多模态（语音、表情、生理信号）情绪识别发展
- **医疗级应用**：英国NHS已将Wysa纳入医保体系，AI心理干预正从消费级向临床级渗透
- **B端市场爆发**：企业EAP（员工帮助计划）成为新增长点，校园心理AI机器人需求激增

**市场规模：**
- 全球数字心理健康市场2025年预计超200亿美元
- 中国心理健康服务市场规模2023年约1500亿元，AI渗透率不足5%，增长空间巨大

**竞争格局：**
- **国际**：Replika（情感陪伴）、Woebot（CBT疗法）、Wysa（综合平台）三足鼎立
- **国内**：尚处早期，EmoGPT（华东师大）、聆心智能（清华背景）等学术派产品崭露头角，缺乏成熟商业产品

### 2.2 用户需求分析

**竞品分析结论：**
- **功能同质化严重**：多数产品停留在聊天机器人+情绪日记层面
- **专业性与温度平衡难**：Woebot过于机械，Replika过于娱乐化
- **中文本土化不足**：国际产品中文理解能力有限，缺乏文化适配

**头脑风暴核心洞察：**
- 用户不仅需要"被倾听"，更需要"被理解"和"被引导"
- 情绪疗愈应该是"轻量级、游戏化、可持续"的日常习惯
- 危机干预（自杀倾向识别）是底线能力，必须建立人工转介机制

## 三、竞品分析

### 3.1 竞品选择
选取国际头部产品 **Woebot**（专业医疗级）和 **Replika**（情感陪伴级）作为核心竞品。

---

### 3.2 竞品概述1：Woebot

**产品名称：** Woebot  
**访问地址：** https://woebothealth.com  
**产品定位：** 基于认知行为疗法（CBT）的临床级AI心理健康助手，由斯坦福心理学家Alison Darcy创立，用户规模超150万

**功能对比：**

| 功能维度     | Woebot                                                 | 本项目（EmoHealer AI）                                       | 对比分析                               |
| ------------ | ------------------------------------------------------ | ------------------------------------------------------------ | -------------------------------------- |
| **核心功能** | CBT结构化对话、情绪追踪、每日检查（Check-in）          | CBT+正念疗法整合、多模态情绪识别（文本+语音+表情）、个性化疗愈方案 | 本项目疗法更多元，支持语音/视频交互    |
| **特色功能** | 循证医学实例丰富、思维记录表（Thought Record）         | AI生成个性化冥想音频、情绪预警雷达、虚拟疗愈陪伴角色         | Woebot偏医疗工具化，本项目重情感化体验 |
| **缺失功能** | 开放式自由对话能力弱、中文支持差、缺乏危机干预人工转介 | 针对中文语境优化、建立"AI-咨询师-医院"三级转介体系           | 本项目填补本土化与闭环服务空白         |

**用户体验：**
- **界面设计**：医疗级简洁风格，以蓝色为主色调，专业但略显冰冷
- **交互体验**：按钮选择式交互为主，用户表达空间有限，流程严谨但机械感强

**竞品优势与劣势：**
- **优势**：CBT疗法专业度极高、循证医学基础扎实、已获FDA突破性医疗器械认定
- **劣势**：2025年6月已停止C端服务（转型B端医疗合作），证明纯工具化模式难以持续；交互体验缺乏温度

---

### 3.3 竞品概述2：Replika

**产品名称：** Replika  
**访问地址：** https://replika.ai  
**产品定位：** AI情感伴侣（AI Companion），主打"永远支持你的AI朋友"，支持友谊/恋爱模式

**功能对比：**

| 功能维度     | Replika                                                | 本项目（EmoHealer AI）                                      | 对比分析                                |
| ------------ | ------------------------------------------------------ | ----------------------------------------------------------- | --------------------------------------- |
| **核心功能** | 开放式对话、情感陪伴、AR互动、记忆用户偏好             | 结构化疗愈对话+开放式陪伴结合、情绪急救工具箱、心理测评体系 | 本项目在陪伴基础上增加专业干预能力      |
| **特色功能** | 虚拟角色养成（等级系统）、社区分享、语音/视频通话      | 基于大模型的认知重构对话、危机识别与干预、疗愈进度可视化    | Replika偏娱乐化，本项目聚焦心理健康疗效 |
| **缺失功能** | 缺乏专业心理疗法体系、无危机识别机制、过度依赖付费订阅 | 免费基础版+增值服务的可持续模式、建立用户安全保护机制       | 本项目解决伦理与商业模式痛点            |

**用户体验：**
- **界面设计**：高度个性化，支持自定义AI形象，视觉风格温暖友好
- **交互体验**：自然语言对话流畅，支持文字/语音/图片/AR多模态，沉浸感强

**竞品优势与劣势：**
- **优势**：AI交互体验业界最佳、用户粘性极高（日均使用时长超40分钟）、情感连接深度强
- **劣势**：专业疗愈属性弱，曾被质疑可能强化用户依赖而非治愈；部分功能涉及伦理争议（恋爱模式）

## 四、需求分析结论

### 4.1 核心用户需求提炼
1. **即时情绪急救**：7×24小时可获得的情感支持（高频需求）
2. **专业且轻松的疗愈**：不希望被当作"病人"，需要"成长伙伴"而非"医生"
3. **隐私绝对安全**：端到端加密，数据本地化存储，匿名使用
4. **可见的改变**：需要数据反馈（情绪曲线、进步报告）获得掌控感
5. **危机时刻兜底**：当AI识别高危风险时，必须有人工介入通道

### 4.2 差异化竞争优势
- **"轻诊疗"定位**：介于Woebot的医疗严肃性和Replika的娱乐陪伴之间，做"专业但有温度"的情绪教练
- **中文语境深度优化**：理解中式家庭关系、职场文化、社交压力等本土化痛点
- **多模态情绪感知**：整合语音语调、面部表情（视频）、生理数据（可穿戴设备）进行情绪评估
- **三级安全体系**：AI预警→在线咨询师干预→线下医院转介的完整闭环

### 4.3 市场挑战
- **监管风险**：心理健康涉及医疗红线，需明确"非医疗器械"定位或申请相关资质
- **用户付费意愿**：国内用户为软件付费习惯尚未成熟，需设计B端（企业/学校）付费模式
- **伦理边界**：避免用户产生过度情感依赖或替代真实人际关系

### 5.2 技术选型

**Web开发技术栈：**
- **后端**：Java + Spring Boot 3.x + MyBatis-Plus + MySQL 8.0
  - Spring Security实现JWT认证与权限控制
  - WebSocket实现实时对话
  - Redis缓存热点数据与对话上下文
- **前端**：Vue 3 + TypeScript + Element Plus + ECharts（数据可视化）
- **基础设施**：Docker容器化部署、Nginx负载均衡、阿里云OSS存储

**人工智能开发技术栈：**
- **核心框架**：Python 3.10 + FastAPI（高性能API服务）
- **大语言模型**： 
  - 基座模型：ChatGLM3-6B/文心一言（中文优化）或 GPT-4 API
  - 微调：LoRA技术基于心理对话数据集微调
- **情绪识别**：
  - 文本：BERT情感分类 + 意图识别
  - 语音：Whisper语音识别 + Wav2Vec情感分析
  - 图像：OpenCV + 轻量级CNN模型（面部表情识别）
- **语音合成**：ChatTTS/Edge-TTS实现温暖自然的AI语音
- **向量数据库**：Milvus存储用户长期记忆与知识库

**数据安全：**
- 敏感数据AES-256加密存储
- 对话内容脱敏处理
- 符合《个人信息保护法》与《数据安全法》要求

---

**项目里程碑建议：**
- **M1（1-2月）**：核心对话引擎开发，基础CBT功能上线
- **M2（3-4月）**：情绪识别算法集成，Web端基础功能完成
- **M3（5-6月）**：危机干预系统、B端管理后台开发
- **M4（7-8月）**：内测优化，获取心理咨询机构合作资源
- **M5（9-12月）**：正式上线，启动企业/高校试点推广

这份任务书结合了最新的行业趋势（Woebot转型B端、多模态情绪识别）和技术可行性，建议重点关注**中文语境优化**和**危机干预安全机制**，这是与海外产品形成差异化的关键。
