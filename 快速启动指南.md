# EmoHealer 快速启动指南

## 一键启动脚本

### Windows 批处理文件
创建 `start_all.bat`:
```batch
@echo off
echo Starting EmoHealer Services...
echo.

echo [1/2] Starting Backend Server...
start "Backend Server" cmd /k "cd backend && python -m uvicorn main:app --host 0.0.0.0 --port 8092 --reload"

echo [2/2] Starting Frontend Server...
start "Frontend Server" cmd /k "cd frontend && python -m http.server 5000"

echo.
echo All services started!
echo.
echo Frontend: http://localhost:5000/emohealer2.html
echo Backend API: http://localhost:8092
echo API Docs: http://localhost:8092/docs
echo.
pause
```

## 手动启动步骤

### 1. 启动后端
```bash
# 方式1: 使用main.py
cd backend
python main.py

# 方式2: 使用uvicorn
python -m uvicorn main:app --host 0.0.0.0 --port 8092 --reload
```

### 2. 启动前端
```bash
cd frontend
python -m http.server 5000
```

### 3. 访问应用
- 前端页面: http://localhost:5000/emohealer2.html
- 后端API: http://localhost:8092
- API文档: http://localhost:8092/docs

## 快速测试

### 方式1: 使用Python脚本

### 方式2: 手动测试

#### 1. 注册用户
```bash
curl -X POST http://localhost:8092/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"123456","nickname":"测试"}'
```

#### 2. 登录
```bash
curl -X POST http://localhost:8092/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"123456"}'
```

#### 3. 发送消息(需要token)
```bash
curl -X POST http://localhost:8092/api/chat/send \
  -H "Content-Type: application/json" \
  -d '{"user_id":1,"message":"你好","emotion":"happy"}'
```

## 服务状态检查

### 检查端口占用
```bash
# 检查后端端口(8092)
netstat -ano | findstr :8092

# 检查前端端口(5000)
netstat -ano | findstr :5000
```

### 停止服务
```bash
# 查找并停止后端进程
taskkill /F /IM python.exe /FI "WINDOWTITLE eq Backend*"

# 查找并停止前端进程
taskkill /F /IM python.exe /FI "WINDOWTITLE eq Frontend*"
```

## 常见问题

### Q1: 后端无法启动
**检查**:
```bash
# 检查Python版本
python --version

# 检查依赖
cd backend
pip list | findstr fastapi
pip list | findstr uvicorn
```

**安装依赖**:
```bash
pip install fastapi uvicorn sqlalchemy pymysql pydantic
```

### Q2: 前端无法访问
**检查**:
- 确认前端服务已启动
- 确认端口5000未被占用
- 检查防火墙设置

### Q3: API返回错误
**检查**:
1. 查看后端日志
2. 访问API文档测试接口
3. 检查数据库连接

### Q4: Token验证失败
**解决**:
- Token通过query参数传递: `?token=xxx`
- 不要使用Authorization头
