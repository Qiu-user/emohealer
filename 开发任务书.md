**项目名称**: EmoHealer AI（基于AIGC的情绪愈疗医生）

**团队规模**: 3人  
**项目类型**: Web应用 + AI服务  
**目标交付**: MVP核心功能

---

## 一、项目概述

### 1.1 项目背景
基于AIGC技术的情绪疗愈平台，为18-35岁用户提供7×24小时专业且温暖的情绪支持服务。解决传统心理咨询费用高、资源分布不均、病耻感阻碍求助等痛点。

### 1.2 核心定位
介于Woebot医疗严肃性和Replika娱乐陪伴之间，做"专业但有温度"的情绪教练。

### 1.3 差异化优势
- 中文语境深度优化（中式家庭关系、职场文化、社交压力）
- 多模态情绪感知（语音语调、面部表情、文本情感）
- 三级安全体系（AI预警→在线咨询师→医院转介）

## 三、技术栈

### 3.1 后端技术
- **框架**: Spring Boot 3.x + MyBatis-Plus
- **数据库**: MySQL 8.0
- **缓存**: Redis（对话上下文）
- **安全**: Spring Security + JWT
- **实时通信**: WebSocket

### 3.2 前端技术
- **框架**: Vue 3 + TypeScript + Vite 5
- **UI组件**: Element Plus + TailwindCSS
- **数据可视化**: ECharts
- **状态管理**: Pinia
- **图标库**: lucide-vue-next

### 3.3 AI服务
- **框架**: Python 3.10 + FastAPI
- **大语言模型**: 文心一言API（或ChatGLM3-6B）
- **情绪识别**: BERT情感分类、Whisper语音识别、CNN表情识别
- **语音合成**: Edge-TTS

### 3.4 基础设施
- **容器化**: Docker + Docker Compose
- **反向代理**: Nginx
- **对象存储**: 阿里云OSS（可选）

#### Day 3-4: 数据库设计与基础服务
**任务ID**: database-design
**负责**: 成员A

**数据库表设计**:
```sql
-- 用户表
CREATE TABLE user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE,
    avatar_url VARCHAR(255),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 对话记录表
CREATE TABLE chat_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT,
    user_message TEXT,
    ai_reply TEXT,
    emotion_type VARCHAR(20),
    is_crisis BOOLEAN DEFAULT FALSE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 情绪日志表
CREATE TABLE emotion_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT,
    emotion_type VARCHAR(20),
    confidence DECIMAL(5,4),
    source VARCHAR(20),
    trigger_context TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 疗愈方案表
CREATE TABLE healing_plan (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT,
    plan_date DATE,
    tasks JSON,
    status VARCHAR(20),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 危机预警表
CREATE TABLE crisis_alert (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT,
    alert_level VARCHAR(20),
    reason TEXT,
    is_handled BOOLEAN DEFAULT FALSE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### Day 5: 后端对话API
**任务ID**: chat-backend
**负责**: 成员A

**API接口**:
```java
@RestController
@RequestMapping("/api/chat")
public class ChatController {
    
    // 发送对话
    @PostMapping("/send")
    public ChatResponse sendMessage(@RequestBody ChatRequest request);
    
    // 获取历史记录
    @GetMapping("/history/{userId}")
    public List<ChatRecord> getHistory(@PathVariable Long userId);
}

@RestController
@RequestMapping("/api/emotion")
public class EmotionController {
    
    // 情绪分析
    @PostMapping("/analyze")
    public EmotionAnalysis analyze(@RequestBody EmotionRequest request);
    
    // 获取情绪趋势
    @GetMapping("/trend/{userId}")
    public EmotionTrend getTrend(@PathVariable Long userId);
}
```

## 七、关键设计规范

### 7.1 API设计规范
- RESTful风格，统一返回格式
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```
- 异常处理统一捕获，记录日志

### 7.2 前端编码规范
- 使用组合式API（Composition API）
- TypeScript类型定义完整
- 组件单文件不超过300行
- TailwindCSS样式优先

### 7.3 AI服务规范
- Prompt模板独立管理
- 模型调用接口可替换
- 异步处理耗时任务

## 八、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| LLM API调用超时 | 对话体验差 | 增加重试机制、超时降级为预设回复 |
| 语音识别准确率低 | 情绪分析偏差 | 多模态融合，文本优先 |
